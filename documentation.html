Project Documentation

This document provides an overview of the file structure and the purpose of each file in the repository.

File Structure

    myDressing/
    ├── dashboard.php
    ├── includes/
    │   ├── auth.php
    │   ├── logs.php
    │   └── db.php
    ├── css/
    │   └── styles.css
    ├── articles.php
    ├── search_articles.php
    ├── add_article.php
    ├── edit_article.php
    └── documentation.html

File Descriptions

dashboard.php

Role: The main dashboard page for the application.

Input: User session data, user role (retrieved from auth.php).

Output: Displays the dashboard with navigation links and cards for managing articles, events, and users (if the user is an admin).

includes/auth.php

Role: Handles user authentication and role management.

Input: User credentials, session data.

Output: Provides methods to check login status and retrieve user roles.

includes/logs.php

Role: Manages logging functionality for the application.

Input: Log messages and events.

Output: Writes logs to a file or database for auditing purposes.

includes/db.php

Role: Provides database connection and query execution functionality.

Input: SQL queries and parameters.

Output: Executes queries and returns results.

css/styles.css

Role: Contains custom styles for the application.

Input: None.

Output: Provides styling for HTML elements.

articles.php

Role: Displays a list of articles and provides options to manage them.

Input: Articles data from the database.

Output: Renders a list of articles with options to add, edit, or delete.

search_articles.php

Role: Allows users to search for articles by title.

Input: Search keyword (via GET request).

Output: Displays a list of articles matching the search keyword.

add_article.php

Role: Provides a form to add new articles to the database.

Input: Article title and description (via POST request).

Output: Adds the new article to the database and redirects to articles.php.

edit_article.php

Role: Provides a form to edit existing articles.

Input: Article ID (via GET request), updated title and description (via POST request).

Output: Updates the article in the database and redirects to articles.php.

Notes

  • Ensure the database schema includes tables for users, articles, and logs.
  • Authentication and database connection are handled through the auth.php and db.php files, respectively.
  • Admin users have additional privileges, such as managing logs and users.
